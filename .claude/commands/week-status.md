# Week Status Report

Generate a comprehensive status report for the current week:

1. **Completed Tasks**: List all completed items from the todo list
2. **Performance Improvements**: Show before/after metrics for any optimizations
3. **Bugs & Blockers**: Identify any issues that need attention
4. **Testing Requirements**: Estimate testing time needed and what needs to be tested
5. **Next Week Preview**: List upcoming tasks from the roadmap
6. **Documentation Updates**: Check if TESTING_FEEDBACK.md or other docs need updates

Format the report as a concise markdown summary suitable for Friday testing sessions.

Include:
- File changes made this week
- Database migrations applied
- API changes or breaking changes
- User-facing feature changes
